## 3. Behavior Changed

**New features:**
- `clang-tidy` CI job runs on Ubuntu with `run-clang-tidy` against `src/core/*.cpp` and `src/telemetry/*.cpp`.
- `asan` CI job builds with `-DAEGIS_CI_BUILD=ON` on Ubuntu Debug, enabling AddressSanitizer (`-fsanitize=address`).
- `AEGIS_CI_BUILD=ON` enables `-Werror` on GCC/Clang and `/WX` on MSVC.
- `tests/data/replay/corpus.tlog` contains encoded MAVLink messages: HEARTBEATĂ—2, ATTITUDE, GLOBAL_POSITION_INT, BATTERY_STATUS, MISSION_CURRENT, STATUSTEXT.
- `test_replay_regression` loads the corpus and verifies all 6 parser paths produce expected state.
- clang-tidy jobs currently use `|| true` to not fail CI (permissive start); warnings are visible in logs.

## 6. Risks and Deviations

**Deviations from plan:**
- clang-tidy jobs use `|| true` (continue-on-error) initially because the existing codebase triggers some warnings from third-party mavlink headers. This is documented with TODO comments in the workflow.

**Challenges:**
- The corpus generator needed to use correct MAVLink API signatures (BATTERY_STATUS `voltages` is `uint16_t*`, MISSION_CURRENT has 7 parameters, STATUSTEXT has 4 parameters).
- Battery voltage units are cV (centivolts), so 1520 cV = 15.2V. The initial corpus used 15200 which produced 152V.
- MSVC `nameless struct/union` warning C4201 from `mavlink_types.h` is third-party noise. Not treated as error in CI (clang-tidy will catch our code, not third_party).

**Known issues / follow-up:**
- clang-tidy findings from third_party headers can be suppressed with `HeaderFilterRegex` tuning.
- Remove `|| true` from clang-tidy jobs once codebase warnings are cleaned up.
- Windows ASan not enabled (MSVC ASan exists but is complex; Linux ASan covers the core logic).
