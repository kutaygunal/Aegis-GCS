# Planner Output — P1-006: CI Hardening

## 1. Summary

Harden the existing GitHub Actions CI with three new quality gates: (1) a permissive clang-tidy static-analysis job on Linux, (2) an AddressSanitizer job on Linux that runs the full test suite under memory-safety instrumentation, and (3) selective warnings-treated-as-errors in CI builds. Additionally, add a replay regression corpus — a committed binary .tlog file exercising all MAVLink parser paths — plus a regression test that replays the corpus and verifies parser coverage.

## 2. Scope Boundary

### In scope:
- Create `.clang-tidy` with a permissive starting config (checks readability, cppcoreguidelines, modernize, performance — no pedantic checks).
- Add `clang-tidy` CI job on Ubuntu that runs on `src/core/` and `src/telemetry/`.
- Add AddressSanitizer (`-fsanitize=address`) CI job on Ubuntu that configures, builds, and runs tests.
- Add selective warnings-as-errors: `-Werror` (GCC/Clang) and `/WX` (MSVC) enabled only in CI via `AEGIS_CI_BUILD=ON` CMake option.
- Create `tests/data/replay/corpus.tlog` — small binary file (≈500 bytes) with MAVLink messages covering all 6 parser paths.
- Add `test_replay_regression` that loads the corpus, replays all messages, and verifies parser state updates.
- Update `.github/workflows/ci.yml` with all new jobs.

### Out of scope:
- MemorySanitizer or ThreadSanitizer (can be added later).
- clang-tidy auto-fixes (manual fixes only).
- Full Windows ASan (MSVC ASan is possible but complex; limit to Linux).
- SonarQube or other external analysis platforms.

### Architecture touch:
- **No** source code changes to `IPlugin`, `TelemetryBus`, `VehicleState`, or business logic.
- Only CI infrastructure, one new test, and optional CMake flags. Zero ABI impact.

- **Content:**

### `tests/data/replay/corpus.tlog`
- **Responsibility:** Binary MAVLink log with messages for all parser paths.
- **Content:** Encoded sequence of:
  1. HEARTBEAT (armed, ACTIVE)
  2. ATTITUDE (roll=0.1, pitch=0.2, yaw=0.3)
  3. GLOBAL_POSITION_INT (lat=377000000, lon=-1220000000)
  4. BATTERY_STATUS (remaining=85, voltage=15.2)
  5. MISSION_CURRENT (seq=3)
  6. STATUSTEXT (severity=WARNING, text="Low battery")
  7. HEARTBEAT (disarmed, STANDBY)
- **Size:** ~400-600 bytes.

### `tests/telemetry/test_replay_regression.cpp`
- **Responsibility:** Load corpus, replay all messages, verify parser state.
- **Key tests:**
  - `CorpusExists` — file is present and non-empty.
  - `AllParserPathsExercised` — replay all messages, verify state reflects each message type.

## 5. Interface Design

### CMake option: `AEGIS_CI_BUILD`
- Default: `OFF`.
- When `ON`, enables `-Werror` (GCC/Clang) or `/WX` (MSVC) for the core and telemetry targets.
- When `ON` + Linux, also enables AddressSanitizer via `-fsanitize=address`.

### CI workflow changes
- **build-linux** job: unchanged (builds and tests on Release).
- **build-windows** job: unchanged.
- **clang-tidy** job (new, Ubuntu):
  - Installs clang-tidy.
  - Runs `run-clang-tidy` or `clang-tidy` on `src/core/*.cpp` and `src/telemetry/*.cpp`.
  - Uses `compile_commands.json` generated by CMake with `-DCMAKE_EXPORT_COMPILE_COMMANDS=ON`.
  - Fails on any clang-tidy warning.
- **asan** job (new, Ubuntu):
  - Configures with `-DAEGIS_CI_BUILD=ON -DCMAKE_BUILD_TYPE=Debug`.
  - Runs `cmake --build` and `ctest`.
  - ASan failures (memory leaks, buffer overflows) fail the CI job.

## 6. State / Data Flow

### CI pipeline
1. Push/PR triggers all jobs in parallel.
2. `build-linux` and `build-windows` verify basic build + test.
3. `clang-tidy` verifies code quality.
4. `asan` verifies memory safety.
5. All four must pass for green CI.

### Regression test
1. Test loads `tests/data/replay/corpus.tlog`.
2. LogReplay reads each MAVLink message and emits `messageReplayed`.
3. MavlinkParser processes each message and updates VehicleState.
4. Test asserts state reflects the final values from the corpus.

## 7. Risk Analysis

| Risk | Likelihood | Mitigation | Fallback |
|---|---|---|---|
| clang-tidy generates false positives | Medium | Start with permissive config; fix real issues; suppress false positives inline | Mark job as `continue-on-error: true` initially |
| ASan finds pre-existing leak | Medium | Run ASan locally first, fix any findings before enabling in CI | Fix leaks in a separate task, then enable ASan |
| Warnings-as-errors breaks Windows | Medium | `/WX` is MSVC equivalent; test with build.bat locally | Remove `/WX` from MSVC if too noisy |
| Corpus binary is large | Low | Corpus is <1KB (6 small MAVLink messages) | Compress or generate at build time |
