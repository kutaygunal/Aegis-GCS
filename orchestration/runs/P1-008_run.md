# Worker Run Report — P1-008: Performant map tile loading

## 1. Task Summary

Implemented a concurrent tile loading pipeline with LRU cache, bounded concurrency, viewport-center priority, cancellation, retry backoff, and metrics observability. The `TileLoader` class replaces ad-hoc network handling with a production-grade system.

## 3. Behavior Changed

**New features:**
- `TileCache` stores tiles as `QImage` in a thread-safe LRU cache with configurable byte-size limit (default 256 MB).
- `TileLoader` checks cache before network; cache hits emit `tileReady` immediately without network.
- Pending queue is sorted by Manhattan distance from viewport center; re-sorted on every `setViewport()`.
- Max concurrent downloads configurable (default 6); excess requests queue.
- `cancelTile()` removes from pending queue; in-flight results are discarded on arrival.
- Retry with exponential backoff for transient failures (max retries = 2, default).
- 4xx errors (permanent) are not retried; `tileFailed` emitted immediately.
- `TileMetrics` observable via `metricsUpdated()` signal: hits, misses, failures, avg latency, pending count, active downloads.

**Changed behaviors:**
- None to existing runtime behavior (TileLoader is new, not yet integrated into map_view plugin).

**Removed behaviors:**
- None.

## 6. Risks and Deviations

**Deviations from plan:**
- Cache stores `QImage` instead of `QPixmap` to avoid `QGuiApplication` requirement in unit tests. `TileLoader` converts `QImage` → `QPixmap` via `QPixmap::fromImage()` when emitting signals.
- No live network integration tests (tests use httpbin.org which requires internet); the `NoRetryFor4xx` test is tolerant of offline environments.
- `TileLoader` is not yet wired into the `map_view` plugin. Integration is deferred to a future UI task to keep this PR focused.

**Challenges:**
- `QMutexLocker<QMutex>` template syntax in Qt6 required removing the `const QMutexLocker&` parameter from `evictIfNeeded()`.
- `QPixmap` requires `QGuiApplication` which isn't available in headless CI. Switching the cache to `QImage` solved this cleanly.
- `QString::arg()` warnings when test URL templates didn't contain `%1/%2/%3` placeholders. Fixed by using query-string placeholders.

**Known issues / follow-up:**
- Pre-existing `ConfigValidatorTest.EmptyConfigAllDefaults` expects `configVersion: 1` but schema now defaults to 2. Fix in dedicated infrastructure cleanup.
- `TileLoader` → `map_view` plugin integration deferred.
- Retry backoff uses `QTimer::singleShot` on the main thread event loop. For very high retry rates, a dedicated timer thread could be considered.

## 7. Review Notes

**Attention items for Reviewer:**
- `aegis_map` is a new static library. The main executable links it; plugins can also link it in future.
- `TileLoader` uses `QNetworkAccessManager` on the main thread. Qt handles actual HTTP on background threads internally, satisfying the "background thread pool" requirement without custom threading code.
- clang-tidy and ASan CI jobs will now also cover `src/map/`.
