# Worker Run Report — P1-002: Structured rotating logs

## 1. Task Summary

Extended the existing `aegis::utils::Logger` with size-based file rotation and a thread-ID-enriched log format. Added `setRotation(qint64 maxSizeBytes, int maxFiles)` to the public API, implemented atomic rotation under the existing mutex, and wired `logging.maxFiles` / `logging.maxSizeBytes` config keys through `Application`. Updated `config/aegis.json` with sensible defaults (5 files, 10 MB each).

## 3. Behavior Changed

**New features:**
- Log files now rotate when they exceed `maxSizeBytes`.
- Rotated backups are named `aegis.log.1`, `aegis.log.2`, etc., up to `maxFiles - 1` backups plus the active file (total ≤ maxFiles).
- Log format now includes a hex thread ID: `[timestamp] [threadId] {level} component: message`.
- Config-driven rotation is active at startup without code changes.

**Changed behaviors:**
- `Application` validates `logging.maxFiles` and `logging.maxSizeBytes`, falling back to defaults if invalid or missing.

**Removed behaviors:**
- None. Existing logger API is fully preserved.

## 6. Risks and Deviations

**Deviations from plan:**
- None. All planned files were created/modified as expected.

**Challenges:**
- Initial rotation logic produced `maxFiles + 1` files because the backup shift loop ran one iteration too many. Fixed by bounding the loop to `m_maxFiles - 2` and deleting `base.(maxFiles - 1)` before shifting.
- Initial `ThreadSafetyStress` test used rotation with a 500-byte limit, which capped total retained size far below the test expectation. Split into two tests: one stress test without rotation and one with rotation that validates count/size bounds instead of total bytes.

**Compromises:**
- None. All acceptance criteria are met without architectural violations.

**Known issues / follow-up:**
- Rotation uses `QFile::rename`, which can fail on Windows if another process (e.g., antivirus, log viewer) holds the file open. The current implementation handles this gracefully because the rename failure is silent and the new file is still opened. A future enhancement could add a small retry loop or use Windows-specific move-file flags.
- Pre-existing test infrastructure failures should be fixed in a separate task (not P1-002).

## 7. Review Notes

**Attention items for Reviewer:**
- `Logger::rotateFiles()` performs all filesystem operations while the logger mutex is held. This is intentionally simple and correct for the expected log volume. If log volume grows extremely high, the mutex hold time could be optimized by pre-computing rename operations outside the lock.
- The `QThread::currentThreadId()` token is formatted as uppercase hex without padding. This is consistent across platforms but may produce variable-width tokens.
- Config defaults in `application.cpp` and `config/aegis.json` must stay in sync. Verify both files if defaults change in the future.
