# Worker Run Report — P1-001: Formal Connection State Machine

## 1. Task Summary

Replaced the implicit boolean connection logic (`m_linkAlive`) in `MavlinkIO` with an explicit `ConnectionStateMachine` supporting six deterministic states: `Disconnected`, `SocketBound`, `VehicleDiscovered`, `HeartbeatAlive`, `Degraded`, `Reconnecting`. Updated the `ConnectionState` enum, `TelemetryBus`, UI `ConnectionBar`, and `Application` wiring to propagate typed state changes.

## 3. Behavior Changed

**New features:**
- Connection now reports 6 explicit states instead of a boolean connected/disconnected.
- `SocketBound` state appears when UDP socket is bound but no packets received yet.
- `VehicleDiscovered` state appears when first MAVLink packet arrives (remote auto-detected).
- `Degraded` state appears after first heartbeat timeout (grace period before full disconnect).
- `Reconnecting` state appears after second consecutive heartbeat timeout.
- All state transitions are logged with reason strings via `qDebug`.

**Changed behaviors:**
- `ConnectionBar` now shows descriptive state names and amber color for intermediate states.
- Dummy telemetry now reports `HeartbeatAlive` instead of generic `Connected`.
- Logger connection messages now report exact state names.

**Removed behaviors:**
- None. The old boolean signal is replaced by the enum signal; consumers are updated.
